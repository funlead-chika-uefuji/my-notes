# JFP1080 出力パイプライン⑥ — JFP1080 専用変換（transform）

**作成日:** 2026-06-24
**対象帳票:** JFP1080 事業別予算査定一覧表（当初）
**プロジェクト:** `/mnt/c/claude/svg-editor/`

---

## 1. Python ファイル名と処理概要

**ファイル名:** `server/app/services/data_prep_reports/JFP1080.py`（中核）
**補助:** `table_lines.py`（`ReportConfig`）／`base.py`（共通ユーティリティ）

**処理概要:**
JFP1080（事業別予算査定一覧表・当初）の **帳票固有の変換ロジック**。3 階層（事業 > 施策 > 施策内訳）のツリーデータを再帰展開し、11 列構成（要求額・1〜4次査定・最終査定・前年度額・差額）の `formData` を生成する。出力パイプライン全体の中で **JFP1080 固有の処理が集約された唯一のファイル**。レイアウト（高さ・座標）は持たず、データ値の整形のみを行う。

---

## 2. ファイル内の処理項目一覧（JFP1080.py）

| 処理プロジェクト名（定義名） | 処理概要 | 処理内容 |
|------|---------|---------|
| モジュール docstring | 帳票仕様の宣言 | 3 階層・11 列構成（header_col1=事業/施策/施策内訳、col2=見積要求額、col3〜7=1〜最終査定、col8=前年度、col9〜11=比較金額/率/予備）を明記 |
| `_NUM_COLS = 11` | 列数定数 | 列数 11 を定義 |
| `PROJECT_BUDGET_ASSESSMENT` | 帳票設定（ReportConfig） | `title="事業別予算査定一覧表(当初)"`、`subtitle_map={}`、`code_id_map={"INITIAL": "JFP1080_..."}`、`item_fields`（col1 は空文字、col2〜11 は 0 を既定）を定義 |
| `resolve_code_id(config, context)` | code_id 解決 | `context["plan_type"]`（既定 `INITIAL`）で `code_id_map` を引く。無ければ `INITIAL` にフォールバック |
| `_fmt_int(n)` | 数値整形 | 空/None→`""`、0→`"0"`、負値→`△{絶対値,}`（三角＋カンマ）、正値→`{値,}`（カンマ区切り）。非数値は `str(n)` |
| `transform(config, context, data)` ★中核 | formData 生成本体 | `context["fiscal_year"]` と `data["tree"]` を取得し、ツリーを `_build` で再帰展開して `{"formData": {...}}` を返す |
| `_build(node, depth)`（transform 内部） | 1 ノード→1 行変換 | 下表参照 |
| ログ出力 | 完了トレース | `logger.info("JFP1080変換完了: 事業=%d", len(items))` |

### `_build(node, depth)` の処理

| 処理プロジェクト名 | 処理概要 | 処理内容 |
|------|---------|---------|
| ラベル生成 | col1 文字列 | `depth` 分の全角空白でインデントし、`code + " " + name` を連結 |
| 査定額の連鎖補完 | 未入力の引き継ぎ | `req=requested_amount`、`a1=assess_1 or req`、`a2=assess_2 or a1`、`a3=assess_3 or a2`、`a4=assess_4 or a3`、`af=assess_final or a4`（未入力時は前段値を継承）。`prev=prev_year or 0` |
| 列マッピング | 11 列への割付 | col1=ラベル, col2=req, col3=a1, col4=a2, col5=a3, col6=a4, col7=af, col8=prev, **col9=af−prev（差額）**, col10/col11=空。各金額は `_fmt_int` で整形 |
| 子ノード再帰 | ネスト表構築 | `node["children"]` があれば `row["items"] = [_build(c, depth+1) ...]` を格納 |
| 最終成形 | formData 組立 | `{"title", "subtitle":"", "fiscal_year":str(...), "report_date":"", "items"}` を `{"formData": ...}` で返す |

## 3. 補助モジュール

### 3.1 `table_lines.py`（JFP1080 が直接 import）

| 処理プロジェクト名（定義名） | 処理概要 | 処理内容 |
|------|---------|---------|
| `ReportConfig`（dataclass） | 帳票設定の型 | `title / subtitle_map / code_id_map / item_fields / default_plan_type("INITIAL")` を保持。JFP1080 はこれで `PROJECT_BUDGET_ASSESSMENT` を生成 |
| `revision_rate(curr, prev)` | 改定率算出（汎用） | `(当−前)/前×100` を四捨五入整数で算出（ゼロ除算回避）。※JFP1080 では未使用 |
| `resolve_code_id`（汎用版） | code_id 解決（汎用） | `plan_type` でマップ参照。※JFP1080 は独自版を使用 |

### 3.2 `base.py`（共通ユーティリティ層／JFP1080 では未使用）

| 処理プロジェクト名（定義名） | 処理概要 | 処理内容 |
|------|---------|---------|
| `to_wareki(fiscal_year)` | 和暦変換 | 西暦年度→「令和7年度」等。0/None は空文字 |
| `format_report_date(iso)` | 帳票日付整形 | ISO 文字列→「2026年2月27日」形式（None は現在時刻） |
| `to_wareki_date(dt)` | 和暦フルデート | 日付→「令和8年6月5日」形式 |
| `now_jst / to_jst` | JST 基準時刻 | 帳票日時を常に JST で扱う |

> 注: JFP1080.py は `report_date=""`・`fiscal_year=str(fiscal_year)` を出力しており、`base.py` の和暦/日付関数は **現状呼び出していない**（他帳票が利用する共通層）。

## 4. 補足（未確定論点との関連）

- `_build` の **「査定額の前段引き継ぎ補完」**（`a1 = assess_1 or req` 等）は、`docs/report/確認依頼_JFP1080_査定額0円の補完問題_5W1H.md` の論点と直結する。査定額が明示的に 0 円のケースと未入力ケースの区別が `or` 演算では付かないため、先方確認の対象。
- レイアウト（タイトル/明細ブロックの高さ）は本ファイルではなく **外部 SVG バインディング（binding_json）** で定義される。
