# JFP1080 出力パイプライン① — 入力スキーマ

**作成日:** 2026-06-24
**対象帳票:** JFP1080 事業別予算査定一覧表（当初）
**プロジェクト:** `/mnt/c/claude/svg-editor/`

---

## 1. Python ファイル名と処理概要

**ファイル名:** `server/app/schemas/print_external_payload/project_assessment.py`

**処理概要:**
業務システム（予算編成）から JFP1080 へ送信される印刷リクエストの **入力検証スキーマ（Pydantic モデル）** を定義するファイル。受信 JSON を `report_type` / `data` / `context` / `sync` の構造で型チェックし、不正な型・必須欠落を API 受信時点で弾く。JFP1080（当初）と JFP2050（補正）の 2 帳票を 1 ファイルで定義しているが、**パイプライン①（JFP1080）が使うのは `Initial` 系クラス**。`context` の共通項目は `_base.py` の `PrintContextBase` を継承して受け取る。レイアウトやデータ加工は行わず、純粋に「入力の型定義・検証」のみを担う。

---

## 2. ファイル内の処理項目一覧

### 2.1 JFP1080（当初）が使用する処理項目 ★主経路

| 処理プロジェクト名（定義名） | 処理概要 | 処理内容 |
|------|---------|---------|
| `JFP1080_CODE_ID` | 帳票識別子の定数定義 | 文字列定数 `"JFP1080_事業別予算査定一覧表(当初)"` を定義。API の URL・code_id 解決に使用 |
| `ProjectAssessmentInitialNode` | 事業ツリー1ノードの型定義 | 3 階層（事業>施策>施策内訳）の 1 ノード。`code/name`（任意）、`level`（必須 int）、`requested_amount`（要求額）、`assess_1〜4`・`assess_final`（各査定額、既定0）、`prev_year`（前年度予算額）、`children`（子ノード配列）を定義 |
| `ProjectAssessmentInitialNode.model_rebuild()` | 自己再帰型の解決 | `children: List["ProjectAssessmentInitialNode"]` という **自己参照（前方参照）** を Pydantic に再構築させ、ネスト構造を有効化 |
| `ProjectAssessmentInitialData` | データ本体の型定義 | `tree: List[ProjectAssessmentInitialNode]` を「事業(level=1)のルートツリー群」として保持 |
| `ProjectAssessmentInitialPrintRequest` | JFP1080 リクエスト全体の型 | 受信ペイロード最上位。`report_type=Literal["project_budget_assessment"]`（固定値で検証）、`data:ProjectAssessmentInitialData`、`context:PrintContextBase`、`sync:bool=True` を束ねる |

### 2.2 同一ファイル内に同居する JFP2050（補正）用の処理項目（参考）

| 処理プロジェクト名（定義名） | 処理概要 | 処理内容 |
|------|---------|---------|
| `JFP2050_CODE_ID` | 帳票識別子の定数定義 | `"JFP2050_事業別予算査定一覧表(補正)"` を定義 |
| `ProjectAssessmentRevisedLine` | 補正版の明細1行の型定義 | JFP2050 は **ツリーではなくフラットな行**。`jigyo/setsu/saimeisai/saisetsu/uchiwake_name`、`pre_amount`（補正前額）、`requested_amount`、`satei_1〜3`・`final_satei`、`post_amount`（補正後額）を定義 |
| `ProjectAssessmentRevisedData` | 補正版データ本体の型 | `lines: List[ProjectAssessmentRevisedLine]` を保持 |
| `ProjectAssessmentRevisedPrintRequest` | JFP2050 リクエスト全体の型 | `report_type=Literal["revised_assessment_by_project"]`、`data`、`context:PrintContextWithStatement`、`sync` を束ねる |

### 2.3 継承元（`_base.py`）— `context` で受け取る共通項目

| 処理プロジェクト名（定義名） | 処理概要 | 処理内容 |
|------|---------|---------|
| `PrintContextBase` | 全帳票必須の context | `frame_id`（予算年度フレームID, 必須）、`fiscal_year`（会計年度・西暦, 必須）を定義。**JFP1080 はこれを直接使用** |
| `PlanType` / `StatementType` | 区分値の型定義 | `PlanType=Literal["INITIAL","REVISED"]`、`StatementType=Literal["P3","P4"]`（収益的/資本的収支） |

---

## 3. 補足

- JFP1080 の `context` は `PrintContextBase`（`frame_id` と `fiscal_year` のみ必須）であり、`plan_type` は含まない。そのため下流（⑥ `JFP1080.py` の `resolve_code_id`）では `context.get("plan_type", "INITIAL")` の既定値 `INITIAL` で code_id が解決される。
- このファイルは **検証専用** であり、数値整形・階層展開・高さ等のレイアウトは一切行わない（それぞれ⑥ `JFP1080.py`、外部 SVG バインディングの担当）。
