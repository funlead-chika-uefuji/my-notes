## 第1章

| 第1章：はじめに（共通方針） |   |   |
| --- | --- | --- |
|   |   |   |
| ■ | 1.1 ガイドラインの目的 |   |
|   |   |   |
|   | 開発者・保守担当者間で設計思想を共有し、ユーザーにとってどの帳票を開いても操作や見方に統一感のあるインターフェースを提供する。 |   |
|   | 公営企業会計システムのリニューアル開発であるため、現行帳票の見た目は現行に近いものを採用することで例規集への影響を最小限に留め、 |   |
|   | 新機能開発に伴う項目追加や現行レイアウトの視認性の低さや出力動線の共通化を行うことで改善を行う。 |   |
|   |   |   |
| ■ | 1.2 改善方針 |   |
|   |   |   |
|   | ① | アクセシビリティの向上：誰にとっても使いやすく、情報を正しく読み取れる設計をする（UDフォント、コントラスト確保等）。 |
|   | ② | 帳票レイアウトの最適化：情報の優先順位を整理し、視線誘導に基づいた効率的な配置への再構成を行う。 |
|   | ③ | 網掛けの改善：視認性を高めるための適切な網掛けルールを適用する（濃淡の調整、行・列の区分）。 |
|   | ④ | 出力形式の拡充と柔軟化：PDF、Excel、CSV等、用途に応じた最適なデータ出力機能を提供する。 |
