# JFP1080 出力パイプライン③ — 出力統括（create_print_job）

**作成日:** 2026-06-24
**対象帳票:** JFP1080 事業別予算査定一覧表（当初）
**プロジェクト:** `/mnt/c/claude/svg-editor/`

---

## 1. Python ファイル名と処理概要

**ファイル名:** `server/app/api/v1/external/print.py`（関数 `create_print_job`）

**処理概要:**
全帳票共通の **汎用印刷エンドポイント**。受信データの準備（④ `apply_data_prep`）→ テンプレート（SVG バインディング）解決 → PDF 生成（⑦）→ 返却、という出力処理全体を統括する。`sync=true` なら同期で PDF バイナリを直接返し、`sync=false` なら Celery ジョブに投入して `202` を返す。各種異常は専用エラーコード付き HTTP 例外で返す。

---

## 2. ファイル内の処理項目一覧（create_print_job の処理順）

| 処理プロジェクト名（定義名） | 処理概要 | 処理内容 |
|------|---------|---------|
| `create_print_job(request, db, api_key)` | エンドポイント本体 | `PrintRequest` を受け、同期/非同期で PDF を生成・返却するメイン関数 |
| json_data 構築 | 入力データの取得 | `request.formdata_file` があれば `_load_formdata_file` で読込、なければ `request.form_data` を `{"formData": ...}` に包む |
| データ準備呼び出し | code_id 解決＋変換 | `apply_data_prep(json_data, request.code_id)`（④）を実行。返った `resolved_code_id` が異なれば `request` を `model_copy` で更新（`"auto"` 対応） |
| テンプレート存在検証 | バインディング有無確認 | `TemplateService(db).get_svgbinding_by_code_id(...)` が無ければ `404 TEMPLATE_NOT_FOUND` |
| 公開バインディング解決 | 印刷対象 Ver の決定 | `resolve_print_binding(code_id, allow_head_fallback=True)` で公開期間有効な Ver を取得。無ければ `404 TEMPLATE_NOT_PUBLISHED` |
| binding_json 検証 | 設定完了チェック | `svgbinding.binding_json` が空なら `400 TEMPLATE_NOT_READY` |
| バッチ分岐 | 複数帳票一括出力 | `request.batch_items` があれば `_handle_batch_print(request, svgbinding)` に委譲 |
| データ必須チェック | 入力欠落の検出 | `json_data` が無ければ `400 NO_DATA`（form_data / formdata_file / batch_items のいずれか必須） |
| `generate_job_id()` | ジョブ ID 採番 | 一意のジョブ ID を発行 |
| 同期モード（sync=true） | PDF 直接返却 | `convert_to_pdf(binding_json, json_data, output_filename)`（⑦）で PDF 生成。`Response(media_type="application/pdf")` で返却（ファイル名は `{code_id}.pdf` を UTF-8 エンコード）。失敗時 `500 PDF_GENERATION_FAILED` |
| 非同期モード（sync=false） | ジョブ投入 | `set_print_job` でステータス登録（24h 有効）、`print_pdf_task.delay(...)` で Celery 投入、`PrintResponse`（job_id・status_url）を `202` で返却 |
| `_handle_batch_print`（参考） | バッチ出力処理 | バッチ項目ごとにファイル名を割り当て Celery タスクへ投入 |

---

## 3. 補足

- JFP1080 は通常 `sync=true`（①スキーマの既定値）で呼ばれるため、**同期モードで PDF バイナリが直接返る**経路を通る。
- このファイルは帳票非依存。JFP1080 固有処理は④以降の `apply_data_prep` → `transform_report` → `JFP1080.transform` で行われる。
