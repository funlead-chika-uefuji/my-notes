# JFP1080 出力パイプライン④ — 形式判定・変換ディスパッチ（apply_data_prep）

**作成日:** 2026-06-24
**対象帳票:** JFP1080 事業別予算査定一覧表（当初）
**プロジェクト:** `/mnt/c/claude/svg-editor/`

---

## 1. Python ファイル名と処理概要

**ファイル名:** `server/app/services/data_prep.py`（関数 `apply_data_prep`）

**処理概要:**
受信した `form_data` の形式を自動判定し、必要に応じて `formData` 形式へ変換する **データ準備のディスパッチ層**。入力には 3 形式（① report_type ベース、② 統一フォーマット fixed+tables、③ formData 直渡し）があり、それぞれ適切な変換へ振り分ける。JFP1080 は **① report_type ベース**に該当し、⑤ `transform_report` を呼んで `code_id` 解決と `formData` 変換を行う。

---

## 2. ファイル内の処理項目一覧（apply_data_prep の処理順）

| 処理プロジェクト名（定義名） | 処理概要 | 処理内容 |
|------|---------|---------|
| モジュール docstring | 役割宣言 | 「form_data の形式を自動判定し、必要に応じて formData 形式に変換する」と明記 |
| import | 依存取り込み | `from app.services.data_prep_report_transformer import is_report_format, transform_report`（⑤） |
| `apply_data_prep(json_data, code_id)` | エントリ関数 | 戻り値 `(変換後 json_data, 解決後 code_id)` を返す |
| formData 取り出し | 入力の抽出 | `form_data = json_data.get("formData", {})` |
| 分岐① report_type 形式 ★主経路 | 帳票変換へ委譲 | `is_report_format(form_data)` が真 → `transform_report(form_data)` で `(resolved_code_id, transformed)` 取得。`code_id` 未指定/`"auto"` なら `resolved_code_id` を採用し `(transformed, code_id)` を返す（JFP1080 はここ） |
| 分岐② 統一フォーマット | 別変換へ委譲 | `fixed+tables` 形式なら `transform_to_formdata(form_data)` で変換して返す |
| 分岐③ formData 直渡し | 無変換通過 | 上記いずれでもなければ `json_data` をそのまま返す（数値整形のみ） |

---

## 3. 補足

- JFP1080 のリクエストは `report_type="project_budget_assessment"` を持つため、必ず **分岐①** を通り、`code_id`（`"auto"` 含む）が `JFP1080_事業別予算査定一覧表(当初)` に解決される。
- このファイルは「どの変換ルートに乗せるか」を決めるだけで、帳票固有の変換ロジックは⑤→⑥に委譲する。
